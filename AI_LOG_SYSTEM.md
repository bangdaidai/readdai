# AI日志系统使用说明

## 功能概述

AI日志系统允许你在应用内直接查看所有AI相关功能的运行日志，无需通过Logcat。这对于调试和问题排查非常方便。

## 如何访问AI日志

### 方法1：通过设置页面

1. 打开应用
2. 进入 **设置** → **AI设置**
3. 找到 **"AI日志"** 选项
4. 点击进入即可查看所有AI日志

### 方法2：直接启动

```kotlin
val intent = Intent(context, AiLogActivity::class.java)
startActivity(intent)
```

## 日志查看器功能

AI日志查看器提供以下功能：

### 1. 实时查看日志
- 显示所有AI相关的操作日志
- 包括时间戳、日志级别、标签和消息内容
- 自动滚动到最新日志

### 2. 刷新日志
- 点击工具栏的 **刷新** 按钮
- 重新加载最新的日志内容

### 3. 复制日志
- 点击工具栏的 **复制** 按钮
- 将所有日志复制到剪贴板
- 方便分享给他人或粘贴到其他应用

### 4. 清空日志
- 点击工具栏的 **清空** 按钮
- 确认后可以清除所有日志
- ⚠️ 此操作不可恢复

### 5. 导出日志（开发中）
- 可以将日志文件导出或分享到其他应用

## 日志信息

日志查看器顶部会显示：
- **日志大小**：当前日志文件的大小（B/KB/MB）
- 当日志超过1MB时会自动清空

## 日志格式

每条日志包含以下信息：

```
[时间戳] [日志级别] [标签] 消息内容
```

示例：
```
[2024-01-15 10:30:45.123] [INFO] [AI按钮] 点击AI按钮: book=xxx, chapterIndex=5
[2024-01-15 10:30:45.234] [INFO] [章节内容] 获取章节内容成功: length=3456, title=第五章 xxx
[2024-01-15 10:30:45.345] [DEBUG] [AI菜单] 显示AI菜单: chapterContent长度=3456
```

## 日志级别

- **DEBUG**：调试信息，详细的执行过程
- **INFO**：一般信息，正常的操作流程
- **WARNING**：警告信息，可能的问题但不影响功能
- **ERROR**：错误信息，功能执行失败

## 记录的日志内容

目前AI日志系统记录以下内容：

### 1. AI按钮操作
- 点击AI按钮
- 书籍信息
- 当前章节索引

### 2. 章节内容获取
- 获取章节内容的结果
- 内容长度
- 章节标题
- 获取失败的错误信息

### 3. AI菜单显示
- 菜单显示时的内容长度

### 4. AI技能执行
- 执行的技能ID
- 传入的内容长度
- 变量映射信息
- 技能找不到的错误

### 5. 前情提要功能
- 开关状态检查
- 当前章节索引
- 前文内容获取情况
- 弹窗显示状态

## 技术实现

### 核心组件

1. **AiLogManager** - 日志管理器
   - 位置：`io.legado.app.help.ai.AiLogManager`
   - 功能：记录和管理AI日志

2. **AiLogActivity** - 日志查看界面
   - 位置：`io.legado.app.ui.ai.AiLogActivity`
   - 功能：显示和操作日志

3. **日志文件**
   - 位置：`/data/data/io.legado.app/files/ai_logs/ai_log.txt`
   - 格式：纯文本文件
   - 大小限制：1MB（自动清理）

### 使用方法

在代码中记录日志：

```kotlin
// 记录普通日志
AiLogManager.log(
    AiLogManager.LogLevel.INFO,
    "标签",
    "日志消息"
)

// 记录带异常信息的日志
AiLogManager.log(
    AiLogManager.LogLevel.ERROR,
    "标签",
    "错误消息",
    exception
)
```

### 初始化

AI日志系统在应用启动时自动初始化（在 `App.kt` 的 `onCreate` 方法中）：

```kotlin
AiLogManager.init(this)
```

## 常见问题

### Q1: 为什么看不到日志？
A: 确保你已经执行过AI相关的操作，比如点击AI按钮、使用AI功能等。

### Q2: 日志太多怎么办？
A: 可以点击 **清空** 按钮清除所有日志，或者等待日志超过1MB后自动清理。

### Q3: 如何分享日志给开发者？
A: 点击 **复制** 按钮，然后粘贴到聊天软件或邮件中发送给开发者。

### Q4: 日志会影响性能吗？
A: 不会。日志写入是异步的，而且文件大小有限制，不会影响应用性能。

### Q5: 可以导出日志文件吗？
A: 目前导出功能还在开发中，暂时只能复制文本内容。
