# AI服务商模型管理功能改进

## 改进概述

参考 archive 项目的设计，为 readdai 项目添加了完整的 AI 服务商模型管理功能。现在一个服务商可以保存多个模型配置，用户可以灵活地管理和切换模型。

## 主要变更

### 1. 数据模型新增

**新增实体：`AiModelConfig`**
- 位置：`AiEntities.kt`
- 功能：独立存储每个服务商的模型配置
- 字段：
  - `id`: 唯一标识
  - `providerId`: 关联的服务商ID
  - `modelId`: 模型ID（如 gpt-4, claude-3-sonnet）
  - `displayName`: 显示名称（可选）
  - `enabled`: 是否启用
  - `sortOrder`: 排序
  - `createdAt/updatedAt`: 时间戳

### 2. 数据库升级

**数据库版本**: 2 → 3
**新增表**: `ai_models`
**新增DAO方法**:
- `getModelsByProvider(providerId)`: 获取服务商的所有模型
- `getModel(id)`: 获取单个模型
- `getModelByProviderAndId(providerId, modelId)`: 根据服务商和模型ID查询
- `insertModel(model)`: 插入/更新模型
- `deleteModel(id)`: 删除模型
- `deleteModelsByProvider(providerId)`: 删除服务商的所有模型
- `setModelEnabled(id, enabled)`: 启用/禁用模型

### 3. UI功能增强

#### 服务商选项菜单
新增"管理模型"选项，位于"编辑"和"测试连接"之间。

#### 模型管理对话框
- **显示所有已添加的模型**：当前使用的模型会标记 ✓
- **设为当前模型**：快速切换正在使用的模型
- **删除模型**：移除不需要的模型配置
- **智能处理**：删除当前模型时自动选择下一个可用模型

#### 获取模型列表优化
- **添加全部**：一键添加所有获取到的模型
- **选择添加**：可以选择添加全部或单个模型
- **去重处理**：自动过滤已存在的模型
- **自动设置**：如果服务商没有当前模型，自动设置为第一个新添加的模型

#### 服务商列表显示
在服务商名称后显示模型数量，例如：`Claude (5个模型) ✓`

### 4. 数据一致性保障

- **级联删除**：删除服务商时自动删除其所有模型
- **模型清理**：删除当前模型时自动选择下一个
- **避免重复**：添加模型时检查是否已存在

## 使用流程

### 场景1：添加新服务商并获取模型

1. 添加服务商（填写API Key等）
2. 点击"获取模型列表"
3. 选择"添加全部"或"选择模型"
4. 系统自动保存模型到配置

### 场景2：管理已有模型

1. 在服务商列表中点击某个服务商
2. 选择"管理模型"
3. 查看所有已添加的模型
4. 点击模型可：
   - 设为当前模型（如果不是当前）
   - 删除模型

### 场景3：切换模型

1. 进入"管理模型"
2. 点击想要使用的模型
3. 选择"设为当前模型"
4. 完成切换

## 技术细节

### 数据库迁移

由于使用了 `fallbackToDestructiveMigration()`，数据库会自动重建表结构。现有数据不会丢失，因为：
- `ai_providers` 表保持不变
- 新增的 `ai_models` 表初始为空
- 用户需要重新获取模型列表

### 向后兼容

- 保留 `AiProviderEntity.model` 字段作为当前使用的模型
- 旧的单模型配置仍然有效
- 新添加的模型会存储在独立的表中

### ID生成策略

模型ID格式：`model_{providerId}_{modelId}_{timestamp}`
- 确保唯一性
- 便于识别所属服务商
- 包含时间戳便于调试

## 后续优化建议

1. **模型别名**：允许用户为模型设置自定义显示名称
2. **模型分组**：按用途分组模型（如聊天、代码、翻译）
3. **模型测试**：添加快速测试模型连通性的功能
4. **导入导出**：支持模型配置的导入导出
5. **默认模型**：为每个服务商设置默认模型
6. **使用统计**：记录每个模型的使用次数和token消耗

## 注意事项

⚠️ **重要提示**：
1. 首次使用后需要重新获取模型列表
2. 删除服务商会同时删除其所有模型
3. 当前使用的模型被删除时会自动切换到下一个
4. 模型配置保存在本地数据库中，不会同步到云端

## 测试建议

1. 添加一个新服务商
2. 获取模型列表并添加全部
3. 验证模型是否正确保存
4. 尝试切换当前模型
5. 删除部分模型
6. 删除服务商并验证级联删除
