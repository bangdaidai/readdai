# 前情提要功能调试指南

## 问题描述
打开了前情提要开关，但没有显示前情提要弹窗。

## 已修复的问题

### 1. 时序问题（已修复）
**原因**：`checkAndShowPreviousSummary()` 在 `viewModel.initData(intent)` 之后立即调用，但 `initData` 是异步的，此时书籍数据可能还没有加载完成。

**修复**：将 `checkAndShowPreviousSummary()` 移到 `initData` 的 `success` 回调中执行，确保数据加载完成后再检查。

```kotlin
// 修复前
viewModel.initData(intent)
justInitData = true
checkAndShowPreviousSummary()

// 修复后
viewModel.initData(intent) {
    // 数据初始化完成后，检查是否需要显示前情提要
    checkAndShowPreviousSummary()
}
justInitData = true
```

### 2. 添加调试日志
添加了详细的日志输出，方便排查问题：
- 检查开关状态
- 检查当前章节索引
- 检查前文内容是否获取成功

## 触发条件

前情提要弹窗需要同时满足以下条件才会显示：

1. **开关已打开**：设置 → AI设置 → 前情提要 开关为 ON
2. **非首次阅读**：当前章节索引 > 0（不是第一章）
3. **有前文内容**：能够获取到之前章节的内容（至少2000字符）
4. **AI服务可用**：AI服务商已正确配置且可用

## 测试步骤

### 步骤1：确认配置
1. 打开设置 → AI设置
2. 确认"前情提要"开关已打开
3. 确认AI服务商已配置（可以使用任意支持的AI服务）

### 步骤2：准备测试书籍
1. 选择一本已经阅读过的书（不在第一章）
2. 确保之前的章节内容已经下载（可以在离线状态下查看）
3. 建议至少有2-3章的内容

### 步骤3：触发前情提要
1. 从书架进入阅读页面
2. 观察是否有前情提要弹窗出现

### 步骤4：查看日志
如果仍然没有显示，请查看 Logcat 日志，搜索关键词"前情提要"：

```bash
adb logcat | grep "前情提要"
```

或者在 Android Studio 的 Logcat 中过滤"前情提要"。

## 可能的日志输出

### 正常情况
```
D/前情提要检查: autoSummary=true
D/前情提要检查: currentChapter=5, book=xxx
D/前情提要: 开始显示弹窗
D/前情提要: 开始获取前文内容
D/前情提要: 获取到的前文内容长度=1856
```

### 常见问题

#### 1. 开关未打开
```
D/前情提要检查: autoSummary=false
D/前情提要: 开关未打开
```
**解决**：在设置中打开前情提要开关

#### 2. 当前是第一章
```
D/前情提要检查: autoSummary=true
D/前情提要检查: currentChapter=0, book=xxx
D/前情提要: 当前是第一章，不显示
```
**解决**：翻到后面的章节，然后重新进入阅读页面

#### 3. 前文内容为空
```
D/前情提要检查: autoSummary=true
D/前情提要检查: currentChapter=5, book=xxx
D/前情提要: 开始显示弹窗
D/前情提要: 开始获取前文内容
D/前情提要: 获取到的前文内容长度=0
D/前情提要: 前文内容为空，不显示弹窗
```
**解决**：确保之前的章节内容已经下载，可以尝试先浏览一下前面的章节

#### 4. bookChanged 为 false
如果没有看到任何"前情提要"相关的日志，可能是 `bookChanged` 标志没有被设置为 true。

**原因**：如果是同一个 Activity 实例只是暂停后恢复（比如按Home键再回来），`bookChanged` 为 false，不会触发检查。

**解决**：完全退出阅读页面（返回书架），然后重新进入。

## 其他注意事项

1. **首次阅读不会显示**：如果你是从第一章开始阅读新书，不会显示前情提要
2. **需要网络连接**：生成前情提要需要调用AI服务，需要网络连接
3. **内容限制**：最多获取2000字符的前文内容用于生成摘要
4. **性能考虑**：如果前文章节很多，获取内容可能需要一些时间
