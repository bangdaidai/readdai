# dai411项目AI功能交互流程设计

## 一、场景总览

AI功能覆盖以下核心场景：

| 场景 | 入口位置 | 功能 |
|------|----------|------|
| 场景1：阅读中长按文本 | 文本选择菜单 | 解释、分析、翻译 |
| 场景2：阅读中点击AI按钮 | 阅读底部工具栏 | 章节摘要、全书总结等 |
| 场景3：结束阅读时 | 阅读退出时弹窗 | 生成当前章节/剧情总结 |
| 场景4：重新进入阅读 | 进入阅读时弹窗 | 前情提要 |
| 场景5：书库中AI入口 | 书籍详情页 | 书籍问答、总结 |
| 场景6：笔记分析 | 笔记页面 | AI整理笔记 |

## 二、详细交互流程

### 场景1：阅读中长按文本

#### 触发条件
用户长按选中一段文本后弹出菜单

#### 交互流程
```
用户长按文本 → 弹出文本选择菜单
                    │
                    ├── [复制] ──────────→ 复制到剪贴板
                    ├── [高亮] ──────────→ 打开高亮颜色选择
                    ├── [笔记] ──────────→ 打开笔记输入
                    ├── [搜索] ──────────→ 全网搜索内容
                    │
                    └── [AI助手 ▼] ─────→ 展开AI子菜单
                                              │
                                              ├── 解释这段 ──→ 直接显示解释结果
                                              ├── 帮我分析 ──→ 显示分析结果
                                              ├── 续写故事 ──→ AI续写内容
                                              └── 更多选项 ──→ 打开AI对话页
```

```
│  │复制 │ │高亮 │ │笔记 │ │搜索 │          │

│                                                │
```

#### AI处理逻辑
- **解释这段**：调用LLM解释选中内容的含义、背景
- **帮我分析**：从写作手法、人物、情节角度分析
- **续写故事**：基于选中内容进行创意续写（需用户确认不偏离原作）
- **更多选项**：跳转到AI对话页面，附带选中内容作为上下文

### 场景3：结束阅读时（自动生成总结）

#### 触发条件
用户退出阅读页面，或切换到其他书籍

#### 交互流程
```
用户点击返回/退出 → 判断是否显示AI总结
                        │
                        ├── 阅读时长 < 2分钟 → 不显示，直接退出
                        │
                        └── 阅读时长 ≥ 2分钟 → 弹出总结弹窗
                                                  │
                                                  ├── 自动生成当前章节摘要
                                                  │    └→ 显示在弹窗中
                                                  │
                                                  ├── 用户可选择"生成更详细总结"
                                                  │    └→ 后台处理，推送通知
                                                  │
                                                  └── 用户点击"暂不"或继续阅读
                                                       └→ 关闭弹窗
```

#### UI展示
```
┌────────────────────────────────────────────────┐
│                                                │
│           📖 本章小结                          │
│                                                │
│  你刚刚阅读了"第15章 危机"                    │
│                                                │
│  ─────────────────────────────────────────── │
│                                                │
│  本章摘要：                                    │
│  主角在面对突如其来的危机时，展现了           │
│  前所未有的勇气与决心。通过与...              │
│                                                │
│  ─────────────────────────────────────────── │
│                                                │
│         [暂不生成]  [生成更详细]              │
│                                                │

│                                                │
│         [✓] 下次不再提醒   [继续阅读]        │
```

#### 后台处理
- 用户选择"生成更详细"后，后台调用AI生成详细总结
- 生成完成后发送通知，用户可点击查看
- 总结内容保存到笔记，标记为"AI生成"

### 场景4：重新进入阅读（自动前情提要）

#### 触发条件
用户从书架选择一本书进入阅读，且距离上次阅读超过30分钟

#### 交互流程
```
用户进入阅读 → 检查阅读间隔
                    │
                    ├── 间隔 < 30分钟 → 正常进入阅读
                    │
                    └── 间隔 ≥ 30分钟 → 显示前情提要弹窗
                                          │
                                          ├── 显示AI生成的简要回顾
                                          │    └→ 包含：上次阅读位置、上次摘要
                                          │
                                          ├── 用户可选择"跳过"
                                          │    └→ 直接进入阅读
                                          │
                                          └── 用户点击"开始阅读"
                                               └→ 关闭弹窗，进入阅读
```

### 4.2 场景判断逻辑

```kotlin
object AiScenarioDetector {
    
    // 判断是否显示前情提要
    fun shouldShowRecall(lastReadTime: Long): Boolean {
        val interval = System.currentTimeMillis() - lastReadTime
        return interval > 30 * 60 * 1000 // 30分钟
    }
    
    // 判断是否显示退出总结
    fun shouldShowSummary(readDuration: Long): Boolean {
        return readDuration > 2 * 60 * 1000 // 2分钟
    }
    
    // 判断从哪个场景触发
    fun detectScenario(trigger: AiTrigger): AiScenario {
        return when (trigger) {
            is AiTrigger.TextSelected -> AiScenario.TEXT_MENU
            is AiTrigger.ToolbarClicked -> AiScenario.TOOLBAR_MENU
            is AiTrigger.ExitReading -> AiScenario.EXIT_SUMMARY
            is AiTrigger.EnterReading -> AiScenario.RECALL
            is AiTrigger.BookDetail -> AiScenario.BOOK_CHAT
        }
    }
}

enum class AiTrigger {
    TextSelected,     // 长按选中文本
    ToolbarClicked,   // 点击工具栏AI按钮
    ExitReading,     // 退出阅读
    EnterReading,    // 进入阅读
    BookDetail       // 书籍详情页
}

enum class AiScenario {
    TEXT_MENU,       // 文本菜单
    TOOLBAR_MENU,    // 工具栏菜单
    EXIT_SUMMARY,    // 退出总结
    RECALL,          // 前情提要
    BOOK_CHAT        // 书籍对话
}
```

## 五、总结

### 场景入口总览图

```

                    │              阅读器主界面                    │

           │                                │                                │

    AI解释/分析/续写                章节摘要/全书总结/              AI自动生成章节摘要

           │                                                │
           │                                                │
           ▼                                                ▼
    显示前情提要弹窗                             与本书对话/生成总结
```

### 核心设计原则

1. **场景驱动**：不同场景使用不同的触发方式和展示形式
2. **渐进式呈现**：从轻量（文本菜单）到完整（独立页面）
3. **智能触发**：根据用户行为（阅读时长、间隔时间）自动判断
4. **上下文感知**：AI始终知道用户当前在读什么、需要什么

这样设计覆盖了阅读前、中、后的完整流程，让AI功能自然融入阅读体验中。
