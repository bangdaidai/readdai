# 历史对话加载功能测试指南

## 功能说明

点击历史对话列表中的某一项后，应该能够：
1. ✅ 加载该会话的所有消息到聊天界面
2. ✅ 正确显示用户消息和 AI 回复
3. ✅ 关闭历史侧边栏
4. ✅ 可以继续在该会话中发送新消息
5. ✅ 新消息会追加到该会话的历史记录中

## 测试步骤

### 1. 准备测试数据

1. 打开 AI 阅读助手
2. 发送几条消息，确保有完整的对话
3. 等待 AI 回复完成
4. 重复几次，创建多个不同的会话

### 2. 测试加载历史对话

#### 步骤 A：打开历史侧边栏
1. 点击顶部菜单的"历史记录"图标（或侧边栏按钮）
2. 应该看到历史会话列表
3. 每个会话显示：
   - 第一条用户消息的前 30 个字符作为标题
   - 最后更新时间（刚刚/X分钟前/X小时前/日期）

#### 步骤 B：点击历史会话
1. 点击列表中的任意一个历史会话
2. 预期结果：
   - ✅ 历史侧边栏自动关闭
   - ✅ 聊天界面清空并加载该会话的消息
   - ✅ 显示 Toast 提示"已加载历史对话"
   - ✅ 自动滚动到最后一条消息
   - ✅ 用户消息右对齐，AI 消息左对齐
   - ✅ Markdown 格式正确渲染

#### 步骤 C：验证消息内容
1. 检查加载的消息是否完整
2. 检查消息顺序是否正确（按时间顺序）
3. 检查特殊格式（代码块、列表等）是否正确显示
4. 如果有工具调用步骤，检查是否正确显示

### 3. 测试继续对话

#### 步骤 A：发送新消息
1. 在加载的历史会话中输入新问题
2. 点击发送按钮
3. 预期结果：
   - ✅ 新消息添加到对话末尾
   - ✅ AI 开始流式回复
   - ✅ 回复过程中定期保存进度
   - ✅ 回复完成后标记为完成

#### 步骤 B：验证保存
1. 关闭 AI 聊天页面
2. 重新打开
3. 打开历史侧边栏
4. 找到刚才的会话（应该在最顶部）
5. 点击进入
6. 预期结果：
   - ✅ 包含之前所有的历史消息
   - ✅ 包含刚才发送的新消息和 AI 回复

### 4. 测试新建对话

#### 步骤 A：通过菜单新建
1. 点击顶部菜单的"新建对话"
2. 预期结果：
   - ✅ 当前对话清空
   - ✅ 创建新的会话对象
   - ✅ 显示空状态界面
   - ✅ 显示 Toast 提示"已新建对话"

#### 步骤 B：通过侧边栏新建
1. 打开历史侧边栏
2. 点击"新建对话"按钮
3. 预期结果：
   - ✅ 当前对话清空
   - ✅ 创建新的会话对象
   - ✅ 侧边栏关闭
   - ✅ 显示空状态界面

### 5. 边界情况测试

#### 测试 A：空会话
1. 创建一个新会话但不发送任何消息
2. 关闭页面再打开
3. 预期结果：
   - ✅ 不会出现在历史列表中（因为没有消息）
   - 或者显示为"空对话"

#### 测试 B：超长消息
1. 发送一条非常长的消息（超过 1000 字）
2. 查看历史列表
3. 预期结果：
   - ✅ 标题被截断为 30 字符 + "..."
   - ✅ 点击进入后显示完整内容

#### 测试 C：未完成会话
1. 发送消息后立即关闭应用（在 AI 回复前）
2. 重新打开
3. 查看历史列表
4. 预期结果：
   - ✅ 会话保存在历史中
   - ✅ 标记为未完成（completed = false）
   - ✅ 只显示用户消息，没有 AI 回复

#### 测试 D：失败会话

2. 发送消息
3. 等待错误返回
4. 查看历史列表
5. 预期结果：
   - ✅ 会话保存在历史中
   - ✅ 包含错误消息
   - ✅ 标记为未完成

## 常见问题排查

### 问题 1：点击历史会话后没有反应
**可能原因**：
- 适配器没有正确更新
- messages 列表引用问题

**解决方案**：
- 确认 `loadSession()` 中调用了 `updateAdapter()`
- 确认重新创建了适配器实例

### 问题 2：加载后消息显示不完整
**可能原因**：
- ChatMessageItem 构造参数不完整
- 缺少 reasoningContent、toolSteps 等字段

**解决方案**：
- 确认创建 ChatMessageItem 时传入所有必需参数
- 设置默认值：reasoningContent = "", toolSteps = emptyList()

### 问题 3：继续发送消息后没有保存到原会话
**可能原因**：
- currentSession 没有正确设置
- saveDraftSession 没有被调用

**解决方案**：
- 确认 `loadSession()` 中设置了 `currentSession = session`
- 确认 `sendMessage()` 中调用了 `saveDraftSession()`

### 问题 4：新建对话后还在原会话中
**可能原因**：
- 没有创建新的会话对象
- currentSession 还是指向旧的会话

**解决方案**：
- 确认"新建对话"时调用了 `createNewSession()`
- 确认清空了 messages 列表

## 代码关键点

### loadSession() 方法
```kotlin
private fun loadSession(session: AiChatSession) {
    // 1. 设置当前会话
    currentSession = session
    
    // 2. 清空并加载消息
    messages.clear()
    session.messages.forEach { msg ->
        messages.add(ChatMessageItem(
            role = if (msg.type == "human") "user" else "ai", 
            content = msg.content,
            reasoningContent = "",
            toolSteps = emptyList(),
            isExpanded = false,
            isReasoningExpanded = false
        ))
    }
    
    // 3. 重置流式位置
    streamingPosition = -1
    
    // 4. 重新创建适配器
    updateAdapter()
    
    // 5. 更新 UI
    updateEmptyState()
    if (messages.isNotEmpty()) {
        binding.recyclerView.scrollToPosition(messages.size - 1)
    }
    
    // 6. 显示提示
    Toast.makeText(this, "已加载历史对话", Toast.LENGTH_SHORT).show()
}
```

### 关键改进点
1. **重新创建适配器**：调用 `updateAdapter()` 而不是 `notifyDataSetChanged()`
2. **完整参数**：ChatMessageItem 包含所有必需字段
3. **重置状态**：清空 streamingPosition
4. **用户体验**：添加 Toast 提示和自动滚动

## 性能优化建议

1. **懒加载**：如果历史会话很多，可以考虑分页加载
2. **缓存**：缓存最近访问的会话，避免重复解析 JSON
3. **索引**：为会话添加搜索索引，支持快速查找
4. **压缩**：对长时间未访问的会话进行压缩存储
