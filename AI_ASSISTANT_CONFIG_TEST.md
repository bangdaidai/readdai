# AI助手自定义配置功能测试指南

## 测试步骤

### 1. 进入配置界面

1. 打开应用
2. 进入 **设置** → **AI设置**
3. 点击 **AI助手配置**

应该看到三个选项：
- 空状态配置（试试这样问）
- 快捷操作栏 - 无选中文字
- 快捷操作栏 - 有选中文字

---

### 2. 测试空状态配置

#### 2.1 查看默认配置
1. 点击 "空状态配置（试试这样问）"
2. 应该看到4个位置：
   - 位置1: 总结本章
   - 位置2: 人物关系
   - 位置3: 主题解读
   - 位置4: 前情回顾

#### 2.2 编辑单个位置（Skill类型）
1. 点击任意位置（如位置1）
2. 确认 "使用Skill" 单选框被选中
3. 从下拉列表选择另一个Skill
4. 点击 "保存"
5. 返回后应该看到更新后的名称

#### 2.3 编辑单个位置（Custom类型）
1. 点击任意位置
2. 选择 "自定义" 单选框
3. 填写：
   - 名称：我的自定义问题
   - 触发词：请帮我分析这段文字
   - 描述：（可选）
4. 点击 "保存"
5. 返回后应该看到 "我的自定义问题"

#### 2.4 恢复默认
1. 在空状态配置对话框中
2. 点击 "恢复默认" 按钮
3. 应该提示 "已恢复默认"
4. 所有位置恢复为预设的Skill

---

### 3. 测试快捷操作栏配置（无选中文字）

#### 3.1 查看默认配置
1. 点击 "快捷操作栏 - 无选中文字"
2. 应该看到5个位置：
   - 位置1: 智能问答
   - 位置2: 总结本章
   - 位置3: 人物分析
   - 位置4: 主题解读
   - 位置5: 前情回顾

#### 3.2 编辑单个位置
1. 点击任意位置（如位置1）
2. 修改显示名称：我的问题
3. 修改触发词：我想问一个问题
4. 点击 "保存"
5. 返回后应该看到 "我的问题"

#### 3.3 恢复默认
1. 点击 "恢复默认" 按钮
2. 应该提示 "已恢复默认"

---

### 4. 测试快捷操作栏配置（有选中文字）

#### 4.1 查看默认配置
1. 点击 "快捷操作栏 - 有选中文字"
2. 应该看到5个位置：
   - 位置1: 解释这个
   - 位置2: 为什么
   - 位置3: 举个例子
   - 位置4: 换句话说
   - 位置5: 深层含义

#### 4.2 编辑单个位置
1. 点击任意位置
2. 修改显示名称和触发词
3. 点击 "保存"

---

### 5. 测试AiChatActivity中的实际效果

#### 5.1 空状态显示
1. 打开一本书
2. 进入AI对话页面（确保没有历史消息）
3. 应该看到4个卡片，显示你配置的内容
4. 点击任意卡片：
   - 如果是Skill类型：应该直接执行并输出结果
   - 如果是Custom类型：应该填充文本并自动发送

#### 5.2 快捷操作栏（无选中文字）
1. 在AI对话页面，不选中任何文字
2. 输入框上方应该显示5个Chip按钮
3. 显示的是 "快捷操作栏 - 无选中文字" 配置的内容
4. 点击任意Chip：
   - 触发词应该填充到输入框
   - 不会自动发送，需要手动点击发送按钮

#### 5.3 快捷操作栏（有选中文字）
1. 在阅读页面选中一段文字
2. 点击 "AI对话" 或类似按钮
3. 选中的文字应该出现在引用区域
4. 输入框上方的Chip应该自动切换为 "快捷操作栏 - 有选中文字" 配置的内容
5. 点击任意Chip：
   - 触发词填充到输入框
   - 可以编辑后发送

#### 5.4 引用管理联动
1. 添加一个引用（选中文字）
2. 观察快捷操作栏是否切换到 "有选中文字" 模式
3. 移除所有引用
4. 观察快捷操作栏是否切换回 "无选中文字" 模式

---

### 6. 边界情况测试

#### 6.1 空文本验证
1. 编辑快捷操作时，尝试将显示名称或触发词留空
2. 点击 "保存"
3. 应该提示 "显示名称和触发词不能为空"
4. 配置不应该被保存

#### 6.2 特殊字符
1. 尝试在名称和触发词中输入特殊字符（如emoji、中文标点等）
2. 保存后重新打开
3. 应该正确显示和保存

#### 6.3 配置持久化
1. 修改一些配置
2. 完全关闭应用
3. 重新打开应用
4. 配置应该仍然保留

---

## 常见问题

### Q1: 修改配置后没有立即生效？
A: 需要重新进入AI对话页面才能看到更新。如果已经打开了AI对话页面，需要退出后重新进入。

### Q2: Skill类型的空状态卡片点击后没反应？
A: 检查该Skill是否正确配置了`showIn`字段包含`"empty_state"`，并且Skill的instruction模板正确。

### Q3: 快捷操作栏没有显示？
A: 检查布局文件`activity_ai_chat.xml`中是否有`quickPromptsLayout`和`promptsContainer`组件。

### Q4: 配置丢失了？
A: 检查SharedPreferences是否正常写入。可以在Android Studio的Device File Explorer中查看：
```
/data/data/io.legado.app/shared_prefs/
```
查找包含`ai_empty_state_config`和`ai_quick_action`的文件。

## 调试技巧

### 查看日志
在Android Studio的Logcat中过滤标签：
```
AiChatActivity
AiAssistantConfigManager
```

### 查看SharedPreferences
使用ADB命令：
```bash
adb shell run-as io.legado.app cat shared_prefs/io.legado.app_preferences.xml | grep ai_
```

### 重置所有配置
在应用设置中找到 "清除数据" 或在代码中调用：
```kotlin
AiAssistantConfigManager.restoreDefaultEmptyStateConfig(context)
AiAssistantConfigManager.restoreDefaultQuickActionBarConfig(context, false)
AiAssistantConfigManager.restoreDefaultQuickActionBarConfig(context, true)
```
