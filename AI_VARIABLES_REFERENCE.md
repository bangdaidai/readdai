# AI 技能和提示词变量参考指南

## 概述

本文档详细介绍了 AI 技能（Skill）和提示词（Prompt）中可用的变量，帮助用户更好地构建自定义的 AI 功能。

## 技能（Skill）变量

技能系统使用双花括号 `{{variable}}` 格式的变量，支持丰富的上下文信息。

### 书籍相关信息

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{{bookName}}` | 书籍名称 | 《红楼梦》 |
| `{{bookAuthor}}` | 书籍作者 | 曹雪芹 |
| `{{bookIntro}}` | 书籍简介 | 中国古代四大名著之一，描写了贾、史、王、薛四大家族的兴衰... |
| `{{currentChapter}}` | 当前章节号 | 12（表示第12章） |

### 章节相关信息

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{{chapterTitle}}` | 章节标题 | 第五回 贾宝玉神游太虚境 |
| `{{chapterContent}}` | 章节内容（最多2000字符） | "却说那林黛玉耳内听了这话，心中自是喜欢..." |
| `{{content}}` | 内容（同 `{{chapterContent}}`，最多2000字符） | 同上 |
| `{{currentContent}}` | 当前内容（最多1000字符） | 章节内容的精简版 |

### 选中文字相关

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{{selectedText}}` | 用户选中的文字 | "这是一个重要的概念" |
| `{{concept}}` | 概念（同 `{{selectedText}}`） | 同上 |
| `{{contextText}}` | 上下文文字（同 `{{selectedText}}`） | 同上 |

### 前情回顾相关

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{{previousContent}}` | 前情内容 | 之前章节的摘要或用户阅读历史 |

### 问答相关

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{{question}}` | 用户提出的问题 | "这段话表达了什么观点？" |

## 提示词（Prompt）变量

提示词系统使用单花括号 `{variable}` 格式的变量，功能相对简单。

### 基础变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{book.name}` | 书籍名称 | 《红楼梦》 |
| `{book.author}` | 书籍作者 | 曹雪芹 |
| `{selectText}` | 选中文字 | "这是一个重要的概念" |
| `{chapterTitle}` | 章节标题 | 第五回 贾宝玉神游太虚境 |

## 使用示例

### 技能示例

```text
技能名称：人物分析
触发词：分析人物
指令：
你是一个文学人物分析专家。
请分析以下内容中的人物形象：

书籍：{{bookName}}
作者：{{bookAuthor}}

选中内容：{{selectedText}}

请分析：
1. 人物性格特点
2. 在故事中的作用
3. 与其他人物的关系
```

### 提示词示例

```text
提示词名称：解释这段
内容：
请解释以下内容的含义：
{selectText}

结合上下文：
{chapterTitle}
```

## 变量使用技巧

### 1. 条件使用
变量可能为空，建议在技能指令中加入判断逻辑：
```
{{bookName}}（如果存在）否则使用"当前书籍"
```

### 2. 内容截断
- `{{chapterContent}}` 最多2000字符
- `{{currentContent}}` 最多1000字符
- 长内容会被自动截断，避免超出AI模型的上下文限制

### 3. 变量组合
可以在同一指令中组合使用多个变量：
```
请分析《{{bookName}}》第{{currentChapter}}章 "{{chapterTitle}}" 中关于 "{{selectedText}}" 的内容。
```

## 注意事项

### 1. 变量格式
- **技能**：使用双花括号 `{{variable}}`
- **提示词**：使用单花括号 `{variable}`

### 2. 变量可用性
- 某些变量只在特定上下文可用（如 `{{selectedText}}` 需要用户选中文本）
- 空变量会被替换为空字符串

### 3. 性能考虑
- 避免在指令中包含过多不必要的变量
- 长内容变量（如 `{{chapterContent}}`）会影响AI响应速度

### 4. 兼容性
- 变量名区分大小写
- 不要在变量名中添加空格或特殊字符

## 最佳实践

### 1. 清晰的变量说明
在创建技能或提示词时，明确说明所需的上下文，让用户了解何时使用该功能。

### 2. 合理的内容长度
- 使用 `{{currentContent}}` 而不是 `{{chapterContent}}` 进行快速分析
- 对于详细分析，可以使用 `{{chapterContent}}`

### 3. 错误处理
在指令中加入适当的容错逻辑，处理变量为空的情况。

### 4. 用户友好
- 在变量名称后添加中文说明
- 提供使用示例
- 保持指令简洁明了

## 常见问题

### Q: 为什么某些变量为空？
A: 变量为空通常是因为当前上下文中没有对应的数据，比如：
- `{{selectedText}}` 需要用户选中文本
- `{{bookName}}` 在没有打开书籍时为空

### Q: 如何测试变量是否有效？
A: 可以创建简单的测试技能，输出变量内容来验证：
```
技能：测试变量
指令：{{bookName}} - {{chapterTitle}} - {{selectedText}}
```

### Q: 变量替换后内容过长怎么办？
A: 使用截断的变量版本（如 `{{currentContent}}` 而不是 `{{chapterContent}}`），或者在指令中明确指出只需要关键信息。
