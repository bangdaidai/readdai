# AI 助手使用说明

---

## 一、基础信息填写

### API 配置

| 字段         | 说明                                                                                                                                                            |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **API 地址** | OpenAI 兼容接口的完整路径，需以 `/v1/chat/completions` 结尾。例：`https://api.openai.com/v1/chat/completions`。使用第三方转发服务时，将域名替换为对应地址即可。 |
| **API Key**  | 接口鉴权密钥，以 `sk-` 开头。Key 仅保存在本地，不会上传至任何服务器，但请注意防止截图泄露。                                                                     |
| **模型名称** | 填写你所使用的模型标识符，例：`deepseek-v4-flash`、`kimi-k2.5`等，纯对话可以使用比较便宜、响应较快的模型，涉及工具调用时，需要使用支持工具调用的模型。          |

### 头像设置

用户头像和 AI 头像均填写**图片直链 URL**（以 `.jpg`、`.png`、`.webp` 等结尾的链接）。

**推荐图床**：[ImgBB](https://imgbb.com/)——上传图片后复制「直接链接」即可使用。

> ⚠️ **注意**：ImgBB 为第三方服务，本应用不对其安全性、可用性及使用效果作任何保证，请自行评估风险。

### 人设（System Prompt）

在「人设」输入框中填写对 AI 角色的基础描述。例如：

```
你是一位博学的文学顾问，擅长分析小说情节和人物关系，回答时请简洁直接。
```

该内容会附加在每次对话的系统提示词最前方。

## 二、工具调用

开启「工具调用」开关后，AI 可以直接查询和管理你的书架数据，无需你手动告知。

### 目前支持的工具

#### 📖 只读工具（8 个）

| 工具                | 功能                                                             |
| ------------------- | ---------------------------------------------------------------- |
| `get_bookshelf`     | 获取书架书籍列表，可按分组筛选                                   |
| `search_bookshelf`  | 按关键词搜索书架中的书籍                                         |
| `get_book_sources`  | 获取书源列表，可按启用状态/分组筛选，支持分页（每次最多 100 条） |
| `get_rss_sources`   | 获取 RSS 订阅源列表，支持分页                                    |
| `get_reading_stats` | 获取阅读统计（总阅读时长、今日阅读、书籍排名等）                 |
| `get_book_chapters` | 获取指定书籍的章节目录                                           |
| `get_book_groups`   | 获取书架分组列表                                                 |
| `get_source_groups` | 获取书源分组列表                                                 |

#### ✏️ 操作工具（8 个）

| 工具                       | 功能                             |
| -------------------------- | -------------------------------- |
| `update_book_group`        | 将书籍移入指定书架分组           |
| `enable_book_source`       | 启用或禁用指定书源               |
| `enable_rss_source`        | 启用或禁用指定订阅源             |
| `update_book_source_group` | 将书源移入指定分组（替换原分组） |
| `create_book_group`        | 创建新的书架分组（最多 64 个）   |
| `delete_book_source`       | 删除指定书源                     |
| `delete_rss_source`        | 删除指定 RSS 订阅源              |
| `delete_book`              | 从书架移除书籍（不删除本地文件） |

### 注意事项

- 书源数量超过 100 个时，AI 每次只能查询 100 条，会自动分批处理并告知进度。
- 如果你拒绝了某次批量整理操作，AI 会提示你告知如何修改方案。
- 工具调用会消耗 token，请注意控制 token 消耗。
- 有其他工具需求，欢迎来[这里](https://github.com/Jingshiro/legado/issues)提issues

## 三、记忆系统

记忆会附加在每次对话的系统提示词中，让 AI 了解你过去阅读时的讨论内容。

### 如何保存记忆

在聊天界面右上角菜单中点击「**归纳记忆**」：

1. AI 会自动总结当前对话的核心内容
2. 总结结果以「章节范围 + 内容摘要」的形式保存
3. 同一章节范围可以保存多条记忆

> 💡 **提示**：归纳记忆功能仅在阅读界面中可用，从「我的」页面进入的独立模式不支持此功能。

### 如何查看和管理记忆

在设置页面点击「**查看记忆列表**」：

- 可以看到所有已保存的记忆条目（章节范围 + 内容）
- 长按某条记忆可以将其删除

点击「**清空记忆**」将一次性删除所有记忆。

### 记忆长度提示

设置页面会显示当前记忆的字符数。记忆过长会占用大量 Token，建议定期清理不再需要的条目。

## 四、使用技巧

- 在「阅读界面」打开 AI 助手，可通过设置「章节范围」来让 AI 参考指定章节的内容进行分析。
- 整理书架时，可以直接告诉 AI「把科幻类书籍都移到科幻分组」，AI 会一次性生成完整方案供你确认。
- 书源数量很多时，建议先让 AI 查看分组列表，再按分组逐步整理。
