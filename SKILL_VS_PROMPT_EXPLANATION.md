# 技能（Skill）与提示词（Prompt）的关系说明

## 核心概念

### 提示词（Prompt）- 简单模板

提示词是最基础的 AI 交互单元，本质上是一个**文本模板**。

```kotlin
data class AiPromptEntity(
    val id: String,
    val name: String,           // 名称：如"解释这段"
    val content: String,        // 内容：如"请解释以下内容：{selectText}"
    val showIn: String,         // 显示位置
    ...
)
```

**特点：**
- ✅ 结构简单，只有一个主要内容字段
- ✅ 使用简单的 `{variable}` 变量格式
- ✅ 适合快速生成固定格式的提示
- ❌ 没有元数据（分类、描述、示例等）
- ❌ 不能自动执行，需要手动选择

**使用场景：**
- 简单的文本转换任务
- 固定格式的问答
- 快速的文本处理

**示例：**
```
名称：解释这段
内容：请解释以下内容的含义和背景：
{selectText}
```

---

### 技能（Skill）- 结构化的高级提示词系统

技能是一个**完整的、结构化的提示词系统**，包含了执行所需的所有信息。

```kotlin
data class AiSkillEntity(
    val id: String,
    val name: String,                    // 技能名称
    val description: String,             // 技能描述
    val triggerWord: String,             // 触发词/关键词
    val instruction: String,             // 详细指令（系统提示词）
    val category: String,                // 分类：summarizer/explainer/analysis等
    val icon: String?,                   // 图标
    val showIn: String,                  // 显示位置
    val sortOrder: Int,                  // 排序
    val isEnabled: Boolean,              // 是否启用
    val isBuiltin: Boolean,              // 是否内置
    val variables: String,               // 变量列表JSON
    val examples: String,                // 使用示例JSON
    ...
)
```

**特点：**
- ✅ 结构复杂，包含丰富的元数据
- ✅ 使用 `{{variable}}` 格式，支持更多上下文变量
- ✅ 有触发词，可以通过关键词自动识别和执行
- ✅ 支持变量自动替换（书名、章节、选中文字等）
- ✅ 有分类系统，便于组织和管理
- ✅ 可以配置显示位置和排序
- ✅ 提供使用示例，帮助用户理解

**使用场景：**
- 复杂的阅读辅助任务
- 需要上下文的分析任务
- 自动化的内容处理
- 结构化的输出要求

**示例：**
```
名称：章节摘要
描述：用简洁语言总结当前章节的主要内容
触发词：总结本章
分类：summarizer

指令：
你是一个专业的书籍内容总结助手。
请用简洁清晰的语言总结以下章节内容。

要求：
- 用3-5句话概括核心内容
- 保留关键人物和事件
- 不要添加主观评价

章节内容：
{{chapterContent}}

变量：["chapterContent"]
示例：["总结这段内容", "帮我概括本章"]
```

## 两者关系

### 1. 技能包含提示词

技能的 `instruction` 字段本质上就是一个**复杂的提示词**：

```
Skill.instruction = 角色定义 + 任务说明 + 输出要求 + 变量模板
```

### 2. 复杂度对比

| 特性 | 提示词 | 技能 |
|------|--------|------|
| 结构 | 简单（单字段） | 复杂（多字段） |
| 元数据 | 无 | 丰富（分类、描述、示例等） |
| 变量系统 | 简单 `{var}` | 强大 `{{var}}`，支持上下文 |
| 自动执行 | ❌ 需手动选择 | ✅ 可通过触发词自动执行 |
| 上下文注入 | ❌ 有限 | ✅ 自动注入阅读上下文 |
| 适用场景 | 简单任务 | 复杂任务 |

### 3. 进化关系

可以把技能看作是提示词的**进化版本**：

```
提示词 → 添加元数据 → 添加变量系统 → 添加执行逻辑 → 技能
```

## 实际使用对比

### 场景：总结章节内容

#### 使用提示词
```
用户操作：
1. 点击"提示词管理"
2. 选择"章节总结"提示词
3. 手动填入章节内容
4. 发送

提示词模板：
"请总结以下内容：{content}"
```

#### 使用技能
```
用户操作：
1. 点击快捷操作栏的"总结本章"chip
2. 系统自动获取当前章节内容
3. 自动执行并返回结果

技能配置：
- 触发词："总结本章"
- 指令：完整的系统提示词（包含角色、要求、格式等）
- 变量：{{chapterContent}} 自动替换
- 分类：summarizer
```

**优势：**
- ✅ 一键执行，无需手动操作
- ✅ 自动获取上下文（章节内容）
- ✅ 更专业的输出（有详细的指令）
- ✅ 可分类管理，易于查找

## 技术实现差异

### 提示词执行流程
```
用户选择提示词 
→ 替换简单变量 {var}
→ 发送到 AI
→ 返回结果
```

### 技能执行流程
```
用户触发技能（点击或输入触发词）
→ 识别技能 ID
→ 加载技能配置
→ 自动注入上下文变量（书名、章节、选中文字等）
→ 替换复杂变量 {{var}}
→ 构建完整的系统提示词
→ 发送到 AI
→ 返回结果
```

## 设计哲学

### 为什么需要两种系统？

1. **简单 vs 复杂**
   - 提示词：适合快速、简单的任务
   - 技能：适合复杂、需要上下文的任務

2. **灵活性 vs 规范性**
   - 提示词：用户可以随意创建和修改
   - 技能：有规范的结构和分类，便于管理

3. **手动 vs 自动**
   - 提示词：需要用户手动选择和填写
   - 技能：可以自动化执行，提升效率

4. **学习曲线**
   - 提示词：容易上手，适合新手
   - 技能：功能强大，适合高级用户

## 最佳实践

### 什么时候用提示词？
- ✅ 简单的文本转换
- ✅ 固定格式的问答
- ✅ 不需要上下文的 task
- ✅ 快速测试想法

### 什么时候用技能？
- ✅ 需要阅读上下文的 task
- ✅ 复杂的分析任务
- ✅ 需要结构化输出的 task
- ✅ 经常使用的功能（可以配置快捷操作）
- ✅ 需要分类管理的 task

---

## 总结

**一句话概括：**
> 提示词是简单的文本模板，技能是带有元数据和执行逻辑的结构化提示词系统。

**类比：**
- 提示词像是"便签纸"：简单、直接、随手可用
- 技能像是"工作流程"：规范、完整、可自动化

**建议：**
- 新手从提示词开始，熟悉后再使用技能
- 简单任务用提示词，复杂任务用技能
- 经常用的功能做成技能，配置到快捷操作栏
- 内置技能可以作为模板，复制后修改成自己的版本
