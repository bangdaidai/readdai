# AI 技能与提示词可用变量完整参考

## 技能（Skill）可用变量

技能使用 `{{variable}}` 格式的变量，系统会自动替换为实际的阅读上下文数据。

### 书籍信息变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `{{bookName}}` | 书名 | 《三体》 |
| `{{bookAuthor}}` | 作者 | 刘慈欣 |
| `{{bookIntro}}` | 书籍简介 | "文化大革命如火如荼进行的同时..." |
| `{{currentChapter}}` | 当前章节号（数字） | 42 |
| `{{chapterTitle}}` | 章节标题 | "第二十二节 红岸之二" |

### 章节内容变量

| 变量名 | 说明 | 限制 | 示例用途 |
|--------|------|------|----------|
| `{{chapterContent}}` | 当前章节完整内容 | 最多2000字符 | 章节总结、内容分析 |
| `{{content}}` | 当前章节内容（同 chapterContent） | 最多2000字符 | 通用内容处理 |
| `{{currentContent}}` | 当前章节部分内容 | 最多1000字符 | 需要精简内容的场景 |

### 选中文字变量

| 变量名 | 说明 | 使用场景 |
|--------|------|----------|
| `{{selectedText}}` | 用户选中的文字 | 解释、分析选中段落 |
| `{{concept}}` | 概念（同 selectedText） | 概念解释技能 |
| `{{contextText}}` | 上下文文本（同 selectedText） | 提供上下文信息 |

### 其他变量

| 变量名 | 说明 | 使用场景 |
|--------|------|----------|
| `{{previousContent}}` | 之前阅读的内容 | 前情回顾、情节追踪 |
| `{{question}}` | 用户的问题 | 智能问答技能 |

## 提示词（Prompt）可用变量

提示词使用 `{variable}` 格式的变量（单层花括号）。

### 可用变量列表

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `{book.name}` | 书名 | 《活着》 |
| `{book.author}` | 作者 | 余华 |
| `{selectText}` | 选中的文字 | "人是为了活着本身而活着的..." |
| `{chapterTitle}` | 章节标题 | "第一章" |

## 变量使用示例

### 示例 1：章节总结技能

```
你是一个专业的书籍内容总结助手。
请用简洁清晰的语言总结以下章节内容。

书籍：{{bookName}}
章节：{{chapterTitle}}

章节内容：
{{chapterContent}}

要求：
- 用3-5句话概括核心内容
- 保留关键人物和事件
- 不要添加主观评价
```

### 示例 2：概念解释技能

```
你是一个专业的概念解释助手。
请用通俗易懂的语言解释以下概念：

概念：{{concept}}

上下文：
{{contextText}}

请解释：
1. 这个概念的基本含义
2. 在这个语境中的具体含义
3. 如果有相关的例子请举一个
```

### 示例 3：前情回顾技能

```
你是一个阅读记忆助手。
请根据之前阅读的内容，帮助用户回顾：

书籍：{{bookName}}
当前章节：第{{currentChapter}}章 {{chapterTitle}}

之前的阅读内容：
{{previousContent}}

请：
1. 简要回顾之前的关键情节
2. 列出主要人物
3. 说明与当前章节的关联
```

### 示例 4：智能问答技能

```
你是一个阅读问答助手。
用户的问题是：{{question}}

当前阅读内容：
- 书籍：{{bookName}}
- 作者：{{bookAuthor}}
- 章节：{{chapterTitle}}
- 内容：{{chapterContent}}

请根据以上信息回答用户的问题。
如果问题与当前阅读内容无关，请礼貌地说明并尝试回答。
```

### 示例 5：提示词示例

```
请解释以下内容的含义和背景：
{selectText}

这本书是 {book.name}，作者是 {book.author}。
当前章节：{chapterTitle}
```

## 变量使用说明

### 1. 格式区别

- **技能**：使用双花括号 `{{variable}}`
- **提示词**：使用单花括号 `{variable}`

### 2. 自动替换

系统会在发送请求到 AI 之前自动替换这些变量：
- 如果变量有值，替换为实际内容
- 如果变量为空或不存在，替换为空字符串

### 3. 内容长度限制

某些变量有长度限制，以防止超出 AI 模型的上下文窗口：
- `{{chapterContent}}`：最多 2000 字符
- `{{content}}`：最多 2000 字符
- `{{currentContent}}`：最多 1000 字符

### 4. 变量可用性

不是所有变量在所有场景下都有值：
- **阅读页面**：书籍信息、章节内容通常可用
- **选中文字时**：`{{selectedText}}` 相关变量可用
- **无前情时**：`{{previousContent}}` 可能为空

### 5. 最佳实践

✅ **推荐做法：**
- 在指令中明确说明使用的变量
- 对长内容变量（如 chapterContent）给出处理指示
- 提供fallback说明（如果变量为空怎么办）

❌ **避免做法：**
- 不要依赖可能为空的变量而不做检查
- 不要在短上下文中使用过多的长内容变量
- 不要假设所有变量都有值

## 常用变量组合

### 场景 1：基于当前章节的分析
```
变量组合：{{bookName}} + {{chapterTitle}} + {{chapterContent}}
适用技能：章节总结、内容分析、写作手法分析
```

### 场景 2：基于选中文字的解释
```
变量组合：{{selectedText}} + {{contextText}} + {{bookName}}
适用技能：概念解释、句子分析、段落解读
```

### 场景 3：全书级别的讨论
```
变量组合：{{bookName}} + {{bookAuthor}} + {{bookIntro}} + {{currentChapter}}
适用技能：全书总结、主题分析、人物关系梳理
```

### 场景 4：连续性阅读辅助
```
变量组合：{{previousContent}} + {{chapterTitle}} + {{chapterContent}}
适用技能：前情回顾、情节预测、人物追踪
```

### 场景 5：智能问答
```
变量组合：{{question}} + {{bookName}} + {{chapterTitle}} + {{chapterContent}}
适用技能：内容问答、细节查询、理解验证
```

## 常见问题

### Q1: 为什么我的变量没有被替换？

**可能原因：**
1. 使用了错误的格式（技能应该用 `{{}}`，提示词用 `{}`）
2. 变量名拼写错误
3. 当前场景下该变量确实没有值

**解决方法：**
- 检查变量名是否与参考列表一致
- 确认当前场景是否支持该变量
- 查看日志确认变量替换情况

### Q2: 变量内容太长怎么办？

**建议：**
- 使用有长度限制的变量（如 `{{currentContent}}` 而非 `{{chapterContent}}`）
- 在指令中要求 AI 只关注关键部分
- 分段处理长内容

### Q3: 可以自定义变量吗？

**目前不支持**。只能使用系统提供的预定义变量。如果需要更多变量，可以：
1. 在技能指令中手动构造所需信息
2. 联系开发者添加新的变量支持
